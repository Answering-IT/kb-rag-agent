## 🔌 WebSocket API (Client Access)

### Connection Details
- **URL:** `wss://1j1xzo7n4h.execute-api.us-east-1.amazonaws.com/dev`
- **API ID:** `1j1xzo7n4h`
- **Status:** ✅ DEPLOYED & TESTED

### Routes
- `$connect` - Connection handler
- `$disconnect` - Disconnection handler  
- `$default` - Message handler (invokes Agent Core Runtime)

### Message Format
```json
// Request
{
  "question": "What documents do you have?",
  "sessionId": "unique-session-id-min-33-chars"
}

// Response (streamed)
{"type": "status", "message": "Processing..."}
{"type": "chunk", "data": "Here are "}
{"type": "chunk", "data": "the documents..."}
{"type": "complete", "sessionId": "..."}
```

### How it Works
```
Client (WebSocket)
  ↓
API Gateway WebSocket
  ↓
Lambda (message_handler.py)
  ├─ Validates session ID (min 33 chars)
  ├─ Invokes Agent Core Runtime via AWS SDK
  └─ Streams response chunks to client
  ↓
Agent Core Runtime (FastAPI on port 8080)
  ├─ Strand Agent processes question
  ├─ Calls tools (KB search, Project Info)
  ├─ Uses Memory (7-day retention)
  └─ Streams response as x-ndjson
```

## 🧠 Memory Configuration

**Type:** SHORT_TERM  
**Retention:** 7 days  
**Storage:** Agent Core Memory (managed by AWS)

**How it works:**
- Each session has isolated memory
- Conversation context retained for 7 days
- Automatic summarization (raw events only)
- No long-term extraction strategies
